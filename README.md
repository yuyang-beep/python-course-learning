# 皇帝的新装 - 多人游戏系统

一个基于 Firebase 的多人游戏管理和结算系统。支持管理者创建房间、控制游戏流程，玩家加入房间进行投票和计分。

```

                   │                   │
         ┌─────────▼──┐      ┌─────────▼──┐
         │ 管理者端    │      │  玩家端    │

         │ · 结束轮数 │      │· 查看成绩  │

         └────────────┘      └────────────┘
```

## 快速开始

### 前置条件
- Google 账号（用于 Firebase）
- 现代浏览器（支持 ES6+ 和 Firebase SDK）
- 网络连接

### 1. 配置 Firebase

按照 [FIREBASE_SETUP.md](./FIREBASE_SETUP.md) 的步骤：
1. 创建 Firebase 项目
2. 获取 Firebase 配置凭证
3. 在 `game-admin.html` 和 `game-player.html` 中填入配置
4. 设置 Realtime Database

### 2. 启动游戏

#### 管理者端
1. 用浏览器打开 `game-admin.html`
2. 点击 **"创建新房间"** 生成房间代码（如 `AB12CD`）
3. 记下房间代码分享给玩家

#### 玩家端
1. 在不同浏览器或设备打开 `game-player.html`
2. 输入房间代码、玩家名称
3. 选择身份（裁缝/大臣/百姓/孩子）
4. 点击 **"加入房间"**

### 3. 游戏流程

1. **配置参数**（管理者端）
   - 设置基础阈值
   - 调整大臣1、2的阈值修正
   - 勾选孩子是否使用特殊质疑

2. **开启轮数**（管理者端）
   - 点击 **"开启本轮"**
   - 所有玩家看到投票界面

3. **投票**（玩家端）
   - 点击三个投票选项之一：
     - 💎 **质疑** - 对裁缝提出质疑
     - ✋ **附和** - 支持裁缝
     - 🤐 **沉默** - 保持沉默
   - 点击 **"提交投票"**
   - 等待其他玩家

4. **结算**（管理者端）
   - 所有玩家投票后，点击 **"结束本轮"**
   - 查看投票统计和结算结果
   - 玩家看到本轮得分和累计分数

5. **重复** 2-4 步进行 5 轮游戏

## 游戏规则

### 身份与职责

| 身份 | 人数 | 目标 |
|------|------|------|
| 👔 裁缝 | 2 | 骗过其他玩家 |
| 🧑‍⚖️ 大臣 | 2 | 发现裁缝谎言 |
| 👨 百姓 | 3 | 支持大臣 |
| 🧒 孩子 | 1 | 说出真话 |

### 计分规则

#### 质疑成功（质疑票数 ≥ 有效阈值）
- **质疑者得分** = 支持者数 + 沉默者数
  - 如孩子使用特殊质疑，额外加 3 分
- **支持者扣分** = 质疑者数
  - 如有孩子特殊质疑，额外扣 3 分
- **裁缝扣分** = 质疑者数
- **沉默者** 无加扣分

#### 质疑失败（质疑票数 < 有效阈值）
- **裁缝得分** = 质疑者数 + 沉默者数
- **支持者得分** = 质疑者数 + 沉默者数
- **质疑者扣分** = 支持者数
  - 孩子使用特殊质疑失败则不扣分
- **沉默者** 无加扣分

### 有效阈值计算
```
有效阈值 = 基础阈值 + 大臣1调整 + 大臣2调整
```

取值范围：0-8，质疑票数需要 ≥ 有效阈值 才能成功

## 玩家隐私

✅ **玩家端只能看到自己的信息**：
- 自己的投票选择
- 自己的本轮得分变化
- 自己的累计分数

- 其他玩家的实时身份
- 游戏的整体结算细节

管理者端可以看到所有信息用于控制游戏流程。

## 常见问题

### Q: 多少人可以同时游戏？
**A**: Firebase 免费版支持 100 个并发连接。一个房间 8 人（2裁缝+2大臣+3百姓+1孩子），所以最多可以支持十几个房间同时进行。

### Q: 游戏可以中途退出吗？
**A**: 可以。玩家点击"退出房间"按钮会从房间移除。管理者点击"退出房间"会删除整个房间。

### Q: 怎样修改轮数（不是 5 轮）？
**A**: 需要修改代码中 Realtime Database 的监听条件。搜索 `currentRound >= 5` 并修改为需要的轮数。

### Q: Firebase 数据会保留多久？
**A**: 无限期保留，直到手动删除。建议定期清理完成的游戏数据。

### Q: 可以离线游戏吗？
**A**: 不可以。需要网络连接来同步数据。

## 故障排除

### 玩家看不到投票界面
- 检查管理者是否点击了"开启本轮"
- 查看浏览器控制台是否有错误信息
- 刷新页面

### 投票提交失败
- 检查网络连接
- 确保 Firebase 配置正确
- 查看浏览器控制台错误

### 房间代码无法加入
- 确保房间代码大小写正确（自动转大写）
- 房间是否由管理者创建
- 管理者是否已断开连接
