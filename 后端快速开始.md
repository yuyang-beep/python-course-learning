## 🎮 使用流程

### 管理者端操作

1. **连接服务器**
   - 服务器地址已默认填写 `http://localhost:3000`
   - 点击 **"连接到服务器"**
   - 看到 ✅ 已连接提示

2. **创建房间**
   - 点击 **"创建新房间"**
   - 获得房间代码（如 `AB12CD`）

3. **等待玩家加入**
   - 玩家使用房间代码加入
   - 管理者端会显示玩家列表

4. **分配身份**
   - 每个玩家旁边有下拉菜单
   - 选择身份并自动保存

5. **开始游戏**
   - 所有玩家分配完身份后
   - 点击 **"开始游戏"**

6. **控制轮数**
   - 配置参数（基础阈值、大臣调整、孩子特殊）
   - 点击 **"开启本轮"**
   - 看到实时投票统计
   - 点击 **"结束本轮"** 显示结果

### 玩家端操作

1. **输入信息**
   - 服务器地址：`http://localhost:3000`（默认）
   - 房间代码：（输入管理者提供的代码）
   - 玩家名称：（输入你的名字）

2. **加入房间**
   - 点击 **"加入房间"**
   - 等待身份分配

3. **等待游戏开始**
   - 等待管理者分配身份
   - 身份分配后点击 **"准备就绪"**

4. **投票**
   - 管理者开启轮数后显示投票界面
   - 点击三个选项之一（质疑/附和/沉默）
   - 点击 **"提交投票"**

5. **查看结果**
   - 所有玩家投票完成后显示结果
   - 看到本轮得分和累计分数
   - 点击 **"进入下一轮"** 继续

## 🆚 Firebase 版本 vs 服务器版本

| 特性 | Firebase | 服务器版 |
|------|----------|---------|
| 部署 | 无需部署 | 需要部署 |
| 成本 | 免费（小规模） | 仅需服务器成本 |
| 控制 | 受限于 Firebase | 完全自主 |
| 实时性 | 实时 | 实时 |
| 扩展性 | 有限制 | 无限制 |
| 维护 | 无 | 需要维护 |
| 推荐场景 | 测试和小型应用 | 正式应用 |

**建议**：
- 开发/测试 → Firebase 版本
- 正式上线 → 服务器版本

## ⚠️ 常见问题

### Q: npm 命令找不到
**答**：Node.js 未正确安装。重新安装 Node.js，然后重启命令行。

### Q: 端口 3000 已被占用
**答**：
```bash
PORT=8000 npm start
```
然后在客户端中使用 `http://localhost:8000`

### Q: 玩家无法连接
**答**：
1. 确认服务器正在运行
2. 检查客户端的服务器地址是否正确
3. 检查防火墙设置
4. 查看浏览器控制台（F12）的错误信息

### Q: 数据丢失
**答**：目前使用内存存储，服务器重启会丢失数据。生产环境应接入数据库。

### Q: 多设备无法连接
**答**：需要在服务器端配置 IP 地址，而不是 localhost。详见 《后端部署指南.md》
