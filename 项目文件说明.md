## 📄 文件详细说明

### 🎮 游戏文件

#### `game-admin.html` 
**管理者端应用**

**功能**:
- 创建新游戏房间或加入现有房间
- 为玩家分配身份（裁缝/大臣/百姓/孩子）
- 配置每轮游戏参数：
  - 基础阈值（0-8）
  - 大臣1调整（-1/0/+1）
  - 大臣2调整（-1/0/+1）
  - 孩子特殊质疑（打勾/取消）
- 实时监控投票统计
- 查看每轮结算结果
- 显示所有玩家的累计分数

**需要配置**:
- Firebase 配置（见下方）

**使用场景**:
- 由主持人/裁判打开
- 控制游戏流程
- 可以同一设备多次打开或在后台监控

---

#### `game-player.html`
**玩家端应用**

**功能**:
- 加入游戏房间（输入房间代码 + 名字）
- 等待身份分配
- 查看被分配的身份和累计分数
- 参与投票（质疑/附和/沉默）
- 查看本轮得分变化
- 跟踪累计分数

**需要配置**:
- Firebase 配置（与管理者端相同）

**使用场景**:
- 由参与游戏的玩家打开
- 每个玩家一个设备或浏览器标签页
- 玩家只能看到自己的投票和分数（隐私保护）

## 🚀 快速上手流程

### 第一次运行（5-10分钟）

1. **准备工作**
   - 有一个 Google 账号
   - 网络连接正常
   - 文本编辑器（记事本、VS Code 等）

2. **配置 Firebase**
   - 按照 `快速开始.txt` 完成 5 个步骤
   - 或按照 `FIREBASE_配置指南.md` 详细配置
   - 获得 Firebase 配置凭证

3. **填入配置**
   - 在 `game-admin.html` 找到 `const firebaseConfig`
   - 粘贴你的 Firebase 配置
   - 在 `game-player.html` 重复此步骤（使用相同配置）
   - 保存两个文件

4. **测试连接**
   - 浏览器打开 `game-admin.html` → 创建房间 → 记住房间代码
   - 新标签页打开 `game-player.html` → 输入房间代码加入
   - 管理者端应立即看到玩家

### 每次游戏运行流程

**准备阶段**（3-5分钟）
1. 管理者打开 `game-admin.html` 创建房间
2. 每个玩家打开 `game-player.html` 加入房间
3. 管理者为所有玩家分配身份
4. 管理者点击"开始游戏"

**游戏进行中**（每轮 2-3 分钟）
1. 管理者配置参数并开启轮数
2. 玩家看到投票界面，选择投票方式
3. 玩家点击"提交投票"
4. 所有玩家完成投票后，管理者点击"结束本轮"
5. 玩家查看本轮得分
6. 重复 1-5 步进行 5 轮

**游戏结束**（1分钟）
- 5 轮完成后，游戏自动结束
- 显示最终分数
- 可以返回加入新的房间

## 🔐 重要提醒

### 安全事项
⚠️ **不要分享你的 Firebase 配置给不信任的人**
- 配置包含 API Key，理论上可以被人利用
- 生产环境必须配置安全规则
- 测试模式只适合开发，不适合公开使用

### 数据隐私
✅ **玩家隐私保护**
- 玩家只能看到自己的投票和分数
- 无法看到其他玩家的详细信息
- 管理者可以看到所有数据（用于验证结果）

## 🛠️ 常见操作

### 如何重新开始一个游戏？
1. 管理者点击"退出房间"
2. 整个房间和数据被删除
3. 点击"创建新房间"开始新游戏

### 如何让玩家中途退出？
1. 玩家点击"退出房间"
2. 玩家从列表中移除
3. 已投票的数据保留（不影响结果）

### 如何修改游戏轮数（不是5轮）？
在 `game-admin.html` 和 `game-player.html` 中搜索 `currentRound >= 5`，改为你想要的轮数。

### 如何导出游戏数据？
1. 打开 Firebase Console
2. 进入 Realtime Database
3. 选择数据并导出为 JSON

### 如何删除旧游戏数据？
1. 打开 Firebase Console
2. 进入 Realtime Database
3. 右键选择要删除的房间数据
4. 点击"删除"

## 🐛 调试技巧

### 查看浏览器控制台
- **Windows**: 按 F12 或 Ctrl+Shift+I
- **Mac**: 按 Cmd+Option+I

在控制台中你能看到：
- Firebase 连接状态
- 错误信息
- 数据同步日志

### 查看 Firebase 数据
1. Firebase Console → Realtime Database
2. 可以实时查看房间和玩家数据
3. 用于验证数据是否正确同步

### 检查网络连接
1. 在浏览器控制台输入：`firebase.database()`
2. 如果返回对象说明连接成功
3. 如果出错说明 Firebase 配置有问题

## 📊 系统性能

### 支持规模
- **玩家数**: 8人/房间（2裁缝+2大臣+3百姓+1孩子）
- **并发房间**: Firebase 免费版支持 100 个并发连接
- **每轮数据量**: ~1KB

### 延迟
- 通常 < 500ms（取决于网络和地区）
- 测试模式没有限制
- 生产环境有速率限制
